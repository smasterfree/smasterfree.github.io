目标

  • 完成vpc环境下，iaas（云主机，网络，硬盘等）相关的集成测试。
  • 在每个物理计算节点，创建云主机，验证网络连通性，云主机生命周期操作，挂卸载云硬盘等。
  • 网关节点验证
  • 管理操作、运维接口验证

安装与部署

  • 拷贝整个项目到节点机或者lxc上

  • 在节点上安装 virtualenv

1
2
3
4
5
6
7

wget https://github.com/pypa/virtualenv/archive/16.0.0.tar.gz
tar -xf 16.0.0.tar.gz
cd virtualenv-16.0.0
python virtualenv.py my_new_env
. my_new_env/bin/activate

或者直接通过 pip install virtualenv 安装 virtualenv
  • 安装 requirements.txt
1

pip install -r requirements.txt

升级依赖。当测试代码新增依赖时，需要在requirements.in 和 requirements.txt加入对应的依赖库，并且定死版本。

推荐使用pip-tools解析依赖。

运行

执行

回归计算节点

1
2
3
4

pytest -n 24   -v  -s   pytest_case/ -m compute

-n 为并发进程数，也可以设置为auto（进程数和cpu核数保持一致）

测试某一个tag下的所有用例

单独回归云主机测试用例集

1

pytest -n 24  -v  -s  pytest_case/ -m compute

单独跑网络测试集

1

pytest -n 24  -v  -s  pytest_case/ -m net

跑冒烟测试集

1

pytest -n 24  -v  -s  pytest_case/ -m smoke

打出执行最慢的10条用例

1
2
3

pytest -n 24  --durations=10  -v  -s  pytest_case/ -m smoke

-m 只运行mark为smoke的测试用例 （可以在用例上增加 @pytest.mark.smoke进行标记）

指定运行某一个测试用例

使用 -k参数。它会去寻找测试用例中包含 floating_ip 字串的用例。

1

pytest -n 6   -v  -s   pytest_case/ -k floating_ip

pytest日志持久化

将pytest的log输出到stdout，并持久化一份到日志目录

1

pytest -n 24   -v  -s   pytest_case/ -m smoke | tee log/pytest-$(date "+%Y-%m-%d-%H-%M-%S").log

在log目录下，pytest- 前缀的日志为用例成功与否的日志。

另有一份时间戳的日志，记录所有操作，云主机uuid等状态检查项详细日志，重点关注error日志。

plugin not registered

如果运行遇到下面这样的问题

1
2
3
4
5

(mhj)hzhuangzhexiao@cyq9:~/vpc-iaas-py$ pytest  -n 24 -v -s pytest_case/ -m smoke
usage: pytest [options] [file_or_dir] [file_or_dir] [...]
pytest: error: unrecognized arguments: -n pytest_case/
  inifile: None
  rootdir: /home/hzhuangzhexiao/vpc-iaas-py

是因为xdist plugin没有register

通过下面这个命令就可以看到 xdist没有注册为 setuptools registered plugins。

1
2

(mhj)hzhuangzhexiao@cyq9:~/vpc-iaas-py$ pytest --version
This is pytest version 3.8.1, imported from /home/hzhuangzhexiao/mhj/local/lib/python2.7/site-packages/pytest.pyc

workaround可以使用 -p参数

1

pytest -p xdist.plugin  -n 24 -v -s pytest_case/ -m smoke

列出所有用例

1

pytest --collect-only

适配iaas ci report的用法

命令行执行参数增加 –json-report

1

pytest  --json-report  -n 5   -v  -s   pytest_case/ -m compute

执行完成之后，运行脚本。 第一个参数是项目名，-m是metadata info，可以不写，也可以方便 提交任何需要补充信息。

1

python ci-report.py  cns -m cns_test_case

会自动将pytest解析 .report.json 生成结果，然后发送到ci report服务器。

注意依赖两个库， 需要提前装好。

1
2

gitpython==2.1.15
pytest-json-report==1.0.1

配置

env.conf 里面配置回归测试哪个环境。

1
2

[env]
test_env = liantiao.conf
  • /etc/*.conf 下面配置环境回归的具体信息。包括租户认证、计算、网络、镜像等相关配置; 注：如果要回归spot相关功能，将conf文件中 business_type = spot，然后执行compute用例。
  • /ssh_key 目录下放对应云主机的ssh key。
  • /node_list 下放对应环境期望回归的节点信息。注意格式。为AZ:HOST 挂载盘类型 。可以随便找个环境先看一下示例。

注意，node_list里面回归的节点以及挂载云硬盘格式。

 1
 2

dongguan1.perf_vpc2:pubbeta2-nova92.dg.163.org ceph,nbs

这样的配置文件，最终会被转换成如下用例，进行回归测试。

[
   ('dongguan1.perf_vpc2:pubbeta2-nova90.dg.163.org', 'ceph'), 
   ('dongguan1.perf_vpc2:pubbeta2-nova90.dg.163.org', 'nbs'),

   ('dongguan1.perf_vpc2:pubbeta2-nova92.dg.163.org', 'ceph'), 
   ('dongguan1.perf_vpc2:pubbeta2-nova92.dg.163.org', 'nbs')
]

conf文件每一项具体含义及说明，可以参考 xxx.conf.sample.

常见错误

  1. 云主机启动失败，没有active（nova服务没起， nova service-list就能看到down）
  2. 网络相关的出错（proton agent服务异常，ovs相关服务检查）
  3. 心跳上报错误（ceilometer服务异常，没起，或者和libvirt之间启动顺序）
  4. Resize出错（resize规格没有改，可能是因为线上没有资源了，可迁移节点都被disable了等）
  5. Migrate出错（可能是由于rsync网段配置错误，导致没法拷贝）
  6. User role not in qa 测试租户没加qa角色
  7. Image does not belong to this tenant 测试镜像没加这个租户
  8. 创建snat失败。一般是因为没资源了。 proton external-ip-pool-list通过这个接口看下资源
  9. volume 状态不对。Nbs或者lvm volume挂卸载卷异常。一般是nbs agent没有部署，导致没法挂卸载。 去节点上ps aux | grep nbs 看下agent部署了没有
  10. 如果待测节点列表中只有一个节点，或者只有一个vpc+sriov节点，迁移会失败。此种情况可以忽略。
  11. 节点列表中混合了ceph和本地存储的节点（一般都是nlb或者rds的），可能导致迁移失败。 因为目前迁移节点是根据是否为sriov进行划分的。本地盘迁移到ceph节点api会报错。
  12. 创建igw失败。可能是由于Resource IGW agent for VpcPeeringConnectio is exhausted，这种情况需要定期清理资源
